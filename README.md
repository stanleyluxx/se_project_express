# WTWR (What to Wear?): Back End

The back-end project is focused on creating a server for the WTWR application. You’ll gain a deeper understanding of how to work with databases, set up security and testing, and deploy web applications on a remote machine. The eventual goal is to create a server with an API and user authorization.

## Technologies and Techniques

This project utilizes a range of technologies and techniques to build a robust back-end server:

### Technologies

- **Node.js**: JavaScript runtime environment for server-side development.
- **Express.js**: Minimalist web framework for Node.js, used to build RESTful APIs.
- **MongoDB**: NoSQL database for flexible and scalable data storage.
- **Mongoose**: Object Data Modeling (ODM) library for MongoDB, providing schema validation and easy data manipulation.
- **Validator**: Library for string validation and sanitization.
- **ESLint**: Tool for identifying and fixing code quality issues.
- **Prettier**: Code formatter to ensure consistent code style.
- **Nodemon**: Utility that monitors for changes in source code and automatically restarts the server during development.

### Techniques

- **RESTful API Design**: Building APIs following REST principles for stateless communication.
- **User Authentication and Authorization**: Implementing secure user login and access control.
- **MVC Architecture**: Organizing code into Models, Views/Controllers, and Routes for better maintainability.
- **Error Handling**: Centralized error management to provide meaningful responses.
- **Input Validation**: Ensuring data integrity through validation middleware.
- **Testing**: Using Postman for API testing and validation.
- **Code Quality**: Employing linting and formatting tools to maintain clean, readable code.
- **Deployment**: Strategies for deploying web applications on remote machines.
